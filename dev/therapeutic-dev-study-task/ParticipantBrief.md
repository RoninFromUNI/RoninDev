# TaskFlow Debugging Exercise — Participant Brief

## Study Title

**Understanding How Developers Naturally Approach Debugging**

## Purpose

This study investigates the strategies and patterns developers use when
diagnosing and fixing software bugs. You will work through a small Java
application called **TaskFlow** that contains several known bugs of
varying difficulty. Your task is to find and fix each bug.

## Setup Instructions

1. Open this project (`therapeutic-dev-study-task`) in **IntelliJ IDEA**.
2. Ensure the **Therapeutic Dev** plugin is visible in the right-hand
   tool window panel. If it is not visible, go to
   **View → Tool Windows → Therapeutic Dev**.
3. Verify the project compiles by running `Main.java`.

## Task Instructions

1. **Run `Main.java`** and examine the output carefully. You will notice
   incorrect behaviour: tasks being rejected when they should be accepted,
   unexpected exceptions, wrong report totals, incorrect sort order, and
   inconsistent state transitions.

2. **Find and fix the bugs.** Work through the issues in the order they
   appear in the output. Each fix may reveal further bugs downstream.

3. **Record timestamps.** On the provided recording sheet, write down:
   - The **time you start** attempting each bug.
   - The **time you successfully fix** each bug.
   - A brief note on what the bug was and how you fixed it.

4. After fixing each bug, **re-run `Main.java`** to verify your fix and
   observe whether new issues appear.

5. There are **5 bugs** in total. The first few are straightforward; later
   bugs are more complex and may span multiple files. Bug 5 is an optional
   stretch — attempt it if you have time.

## Estimated Time

**60–90 minutes** in total. Work at your own pace. If you get stuck on a
bug for more than 20 minutes, move on to the next one and return later.

## Important Notes

- Do **not** modify `Main.java` — it is the test harness.
- You may add `System.out.println()` statements to help debug.
- You may use IntelliJ's debugger, breakpoints, and step-through features.
- There are no trick questions — every bug has a clear, logical fix.
